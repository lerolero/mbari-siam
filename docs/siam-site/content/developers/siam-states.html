SIAM Instrument Service States

State transitions for SIAM instrument services

Instrument Service Framework: Instrument Service States

SIAM Service State Description
INITIAL Instrument service and device are initializing
OK Instrument is ready to sample
SAMPLING Instrument is acquiring a sample
SLEEPING The instrument service is sleeping for a predetermined amount of time (cpu may safely go to sleep if service is SLEEPING)
SAFE Instrument device is in a state compatible with system power loss. Service does not interact with device while in this state (Few instruments implement Safeable)
SUSPEND Corresponds to a "direct mode"; service will not interact with device while in this state
ERROR Latest sampling attempt encountered an error
SHUTDOWN The instrument service and device are shutting down
