Name

Transfer Data from BIN to primary node

Synopsis

Prior to recovering the MSE SHELF BIN, its data should be transferred to the ship.

For efficiency and data safety, the data should first be copied from the SHELF BIN to the surface via ethernet.

Once it is safely located on the surface, the transfer of the data to the ship can progress concurrently with the recovery of the SHELF BIN.

Connect to primary surface node

Establish radio console to SURFACE

Login to SURFACE via radio console

Modify radio console timeout
TMOUT=3600

Establish CPU lease on SURFACE

a 3600

Connect to BIN

Wake up SHELF BIN

bcastc

Establish lease on SHELF BIN

lease –establish 3600 node1

SSH to SHELF BIN

ssh ops@node1 gosiam

Verify that the SIAM application is not running

ps –ef

look for j9 processes; if there are still many, terminate them

killj9

Prepare to transfer

Make sure that there is enough (> 50%) disk space on /dev/hda1 to store compressed logs

df -Th

Make a directory to store compressed logs

mkdir xfer

stop instrument services (for each instrument)

shutdownPort loc <portName>

Compress the data directories into tar files, e.g.,

tar czvf xfer/mse-shelf-siam1520.tar.gz logs/1520*
tar czvf xfer/mse-shelf-power1554.tar.gz logs/1554*
tar czvf xfer/mse-shelf-seabird1533.tar.gz logs/1533*
tar czvf xfer/mse-shelf-wetlabs1535.tar.gz logs/1535*
tar czvf xfer/mse-shelf-aquadopp1474.tar.gz logs/1474*
tar czvf xfer/mse-shelf-aanderaa1444.tar.gz logs/1444*
tar czvf xfer/mse-shelf-workhorse1525.tar.gz logs/1525*
tar czvf xfer/mse-shelf-msp4301527.tar.gz logs/1527*
tar czvf xfer/mse-shelf-medusa1526.tar.gz logs/1526*

start instrument services

scanPort loc <portName> (suspend/resume/restartPort…resumePort leaves services in SHUTDOWN state)

Transfer files

Transfer the tar files to the SURFACE (log results)

scp xfer/*.tar.gz ops@surface:/mnt/hda/.|tee fileTransfer.out
