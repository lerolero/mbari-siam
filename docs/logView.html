NAME
logView - Display packets from specified SIAM log to stdout
SYNOPSIS 
logView [options] deviceID directory
DESCRIPTION
The logView utility displays packets from the specified SIAM device log, residing in the specified directory. Note that the SIAM "log" for a given device actually consists of two files, a "dat" file and an "idx" (index) file. For example, the log for device ID "1287" consists of the files "1287_0.dat" and "1287_0.idx". To view all packets from device 1287, the user would 'cd' to the directory in which these files reside and type:
  
% logView 1287 .
OPTIONS
  • -utc- Print timestamp in UTC format
  • -type type- Only print packets of the specified type, where type is one of the following: "all", "metadata", "data", or "message".
  • -bin- Print packets in "raw" binary form.
  • -start <timespec> - Do not print packets with created before the specified time, where timespec
  • -stop <timespec- Do not print packets created after the specified time.
Valid timespec formats for the -start and -stop options are as follows:
  • m/d/yyyy - numeric month, day-of-month, year
  • m/d/yyyyTh:m:s - numeric month, day-of-month, year, followed by letter 'T', then hour, minutes, seconds


EXAMPLES

To view all metadata packets from device 1321, when the log files reside in the current directory, the user would type:
  
% logView -type metadata 1321 .
To view all packets created from device 1287, between noon on July 12, 2004 and 3 AM on July 15, when the log files reside in directory ~/logs, the user would type:
  
% logView -start 7/12/2004T12:0:0 -stop 7/15/2004T3:0:0 1287 ~/logs


KNOWN BUGS

The -start and -stop "time window" options only work correctly if all packets in the log are sorted in ascending timestamp order; utilities such as getPackets may generate logs that are not ordered in this way.

ENVIRONMENT

The following environment variables must be set:
  • JAVA_HOME - location of Java installation, e.g. /j9 or /opt/java/jdk1.3.1
  • JAVA - name of the JVM. E.g. on a Sidearm, JAVA should be set to j9. On a workstation with Sun's Java installed, JAVA could be set to java.
  • SIAM_HOME - top of the SIAM directory tree
  • SIAM_CLASSPATH - on a Sidearm, SIAM_CLASSPATH should be set to $JAVA_HOME/lib/jclMax/classes.zip:$JAVA_HOME/lib/prsnlmot.jar:$JAVA_HOME/lib/RXTXcomm.jar:$SIAM_HOME/classes.  On X86 system, SIAM_CLASSPATH should be set to $SIAM_HOME/classes
