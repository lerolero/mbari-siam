NAME
writePuck - write service code and metadata to the pucks non-volatile memory
SYNOPSIS 
writePuck jarFile portName [-id]
DESCRIPTION
The writePuck utility is used to store service code and metadata to a pucks non-volatile memory. To use writePuck attach the puck to the appropriate serial port and power it up. After the puck has been powered up enter the writePuck command followed by the necessary arguments, for example 
$ writePuck Compass-1222.jar /dev/ttySX3
If the -id option is specified only an instrument ID will be written to the puck. The SIAM application will attempt to look for the service code and metadata locally if they are not located in the puck. This utility may be run on the MMC as well as a Windows or Linux PC. Though the utility may be used on the MMC it should not be used while the SIAM application is running. 





ENVIRONMENT

The following environment variables must be set:



    

  • 
JAVA_HOME - location of Java installation, e.g. /j9 or /opt/java/jdk1.3.1
    • 


  • 
JAVA - name of the JVM. E.g. on a Sidearm, JAVA should be set to j9.
On
a workstation with Sun's Java installed, JAVA could be set to java.
    • 


  • 
SIAM_HOME - top of the SIAM directory tree
    • 


  • 
SIAM_CLASSPATH - on a Sidearm, SIAM_CLASSPATH should be set to SIAM_CLASSPATH=$JAVA_HOME/lib/jclMax/classes.zip:$JAVA_HOME/lib/prsnlmot.jar:$JAVA_HOME/lib/RXTXcomm.jar:$SIAM_HOME/classes:$SIAM_HOME/jars/log4j-1.2.8.jar. 
On X86 system, SIAM_CLASSPATH should be set to $SIAM_HOME/classes;$SIAM_HOME/jars/RXTXcomm.jar;$SIAM_HOME/jars/log4j-1.2.8.jar
    •
