NAME
setInstClock - 'synchronize' instrument clock to node clock
SYNOPSIS 
setInstClock nodeHost portName(s)
DESCRIPTION
The setInstClock utility approximately synchronizes an instrument clock with the SIAM node clock. Synchronization accuracy cannot be guaranteed.

Multiple ports may be specified, e.g. 

% setInstClock sidearm6 /dev/ttySX1 /dev/ttySX2 /dev/ttySX3
The portName(s) can be abbreviated as an integer, i.e. "/dev/ttySX1" can be abbreviated as simply "1". Thus the above example is equivalent to 
% setInstClock sidearm6 1 2 3

This utility may be run on the same host as the node, or on any networked host from which the node host is accessible. If run on the node's host, then nodeHost should be specified as "localhost". 






ENVIRONMENT

The following environment variables must be set:



    

  • 
JAVA_HOME - location of Java installation, e.g. /j9 or /opt/java/jdk1.3.1
    • 


  • 
JAVA - name of the JVM. E.g. on a Sidearm, JAVA should be set to j9.
On
a workstation with Sun's Java installed, JAVA could be set to java.
    • 


  • 
SIAM_HOME - top of the SIAM directory tree
    • 


  • 
SIAM_CLASSPATH - on a Sidearm, SIAM_CLASSPATH should be set to $JAVA_HOME/lib/jclMax/classes.zip:$JAVA_HOME/lib/prsnlmot.jar:$JAVA_HOME/lib/RXTXcomm.jar:$SIAM_HOME/classes. 
On X86 system, SIAM_CLASSPATH should be set to $SIAM_HOME/classes
    •
