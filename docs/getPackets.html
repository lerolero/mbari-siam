NAME
getPackets - retrieve packets from specified instrument service on specified host
SYNOPSIS 
getPackets nodeHostname [options] deviceID
DESCRIPTION
The getPackets utility retrieves packets from a specified instrument service, subject to subsampling and filtering specified by options specified on the command line, and writes the packets to a device log in the current directory. The device log consists of two files; the '.dat' file contains the packets, and the '.idx' is an index into the .dat file (providing efficient "random" access to the .dat file). The files' basename consists of  the deviceID with "_0" appended; e.g. the log for device 1473 consists of files 1473_0.dat and 1473_0.idx. If the log files for the specified device already exist, getPackets will append to the log. The logged packets can subsequently be displayed with the logView utility.

NOTE: the instrument service is specified with the integer deviceID, not by port name.

OPTIONS

-between startTime endTime
Retrieve only packets created between startTime and endTime, where times can have format m/d/yyyy, m/dyyyyTh:m, m/d/yyyyTh:m:s. By default, startTime is January 1, 1970 and endTime is the current clock time.
-types type(s)
Retrieve only packets of specified type(s), where type can be 'metadata', 'data', 'message', 'summary', or 'all'. By default, all types are retrieved.
-skip interval
Retrieve only every nth packet, where n=interval + 1. By default, interval is 0, which means don't skip any packets.
-default:
Use any telemety filters already associated with the service when retrieving packets.
ENVIRONMENT
The following environment variables must be set:
  • JAVA_HOME - location of Java installation, e.g. /j9 or /opt/java/jdk1.3.1
  • JAVA - name of the JVM. E.g. on a Sidearm, JAVA should be set to j9. On a workstation with Sun's Java installed, JAVA could be set to java.
  • SIAM_HOME - top of the SIAM directory tree
  • SIAM_CLASSPATH - on a Sidearm, SIAM_CLASSPATH should be set to $JAVA_HOME/lib/jclMax/classes.zip:$JAVA_HOME/lib/prsnlmot.jar:$JAVA_HOME/lib/RXTXcomm.jar:$SIAM_HOME/classes.  On X86 system, SIAM_CLASSPATH should be set to $SIAM_HOME/classes
