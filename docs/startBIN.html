NAME
startBIN - enable BIN 400V power when node reboots
SYNOPSIS 
startBIN [-Vhd][-D -p -P -s -S]
DESCRIPTION
Option Description Default Value
-d debug (read but don't set BIN power) [FALSE]
-D start delay seconds [90]
-H host node
-h print this help message
-p power retries [5]
-s service retries [3]
-P power wait seconds [20]
-S service wait seconds [20]
-V verbose output [FALSE]

Although it has always been standard operating procedure to exercise caution around BIN 400V power, please observe the following precaution/reminder:

CAUTION: BIN 400V power may be turned on at any time. Always treat the BIN as if 400V power is on.

The script is located in siam/utils/ directory and it is called from /etc/init.d/siamapp (the script that automatically starts SIAM when the node boots).

For MUCE, startBIN is called using default parameters and the -V (verbose output). The output from the most recent startBIN invocation is directed to /tmp/startBIN.out.
To disable startBIN, the line in siamapp that calls startBIN may be commented out.

startBIN waits for an optional start delay (so the power can service can start), then tries several (service retries) times to detect the power service, waiting (service wait seconds) between retries to determine what port it is running on. If it successfully determines the power service port, it then it tries several (power retries) times to turn on the BIN power, waiting (power wait seconds) between retries.

startBIN returns 0 upon success, 1 on failure.


ENVIRONMENT

The following environment variables must be set:
  • JAVA_HOME - location of Java installation, e.g. /j9 or /opt/java/jdk1.3.1
  • JAVA - name of the JVM. E.g. on a Sidearm, JAVA should be set to j9. On a workstation with Sun's Java installed, JAVA could be set to java.
  • SIAM_HOME - top of the SIAM directory tree
  • SIAM_CLASSPATH - on a Sidearm, SIAM_CLASSPATH should be set to $JAVA_HOME/lib/jclMax/classes.zip:$JAVA_HOME/lib/prsnlmot.jar:$JAVA_HOME/lib/RXTXcomm.jar:$SIAM_HOME/classes.  On X86 system, SIAM_CLASSPATH should be set to $SIAM_HOME/classes
