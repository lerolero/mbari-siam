Service jar files and PUCKs

catPuck - quickly identify the cotents of a puck 
catPuck portName
puckit - create "jar file" for instrument service 
puckit serviceClassName propertiesFile jarFile
readPuck - extract the contents of the puck and store it in a local file

writePuck - write service code and metadata to the pucks non-volatile memory 
writePuck jarFile portName [-id]
