NAME
readPuck - extract the contents of the puck and store it in a local file
SYNOPSIS 
readPuck portName
DESCRIPTION
The readPuck utility can be used to extract the contents of a puck and store it in a local file 'puck.jar'. To use readPuck attach the puck to the appropriate serial port and power it up. After the puck has been powered up enter the readPuck command followed by the necessary arguments, for example 
$ readPuck /dev/ttySX3
This utility may be run on the MMC as well as a Windows or Linux PC. Though the utility may be used on the MMC it should not be used on a port that has an active instrument service. 





ENVIRONMENT

The following environment variables must be set:



    

  • 
JAVA_HOME - location of Java installation, e.g. /j9 or /opt/java/jdk1.3.1
    • 


  • 
JAVA - name of the JVM. E.g. on a Sidearm, JAVA should be set to j9.
On
a workstation with Sun's Java installed, JAVA could be set to java.
    • 


  • 
SIAM_HOME - top of the SIAM directory tree
    • 


  • 
SIAM_CLASSPATH - on a Sidearm, SIAM_CLASSPATH should be set to SIAM_CLASSPATH=$JAVA_HOME/lib/jclMax/classes.zip:$JAVA_HOME/lib/prsnlmot.jar:$JAVA_HOME/lib/RXTXcomm.jar:$SIAM_HOME/classes:$SIAM_HOME/jars/log4j-1.2.8.jar. 
On X86 system, SIAM_CLASSPATH should be set to $SIAM_HOME/classes;$SIAM_HOME/jars/RXTXcomm.jar;$SIAM_HOME/jars/log4j-1.2.8.jar
    •
