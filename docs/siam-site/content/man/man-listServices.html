Name

listServices

Synopsis

listServices nodeURL

Description

The listServices utility lists all the services that have been added to a node using addService. The listing includes the universally unique identifiers (UUID) for each service.

addService is used to register the UUID of instrument services for instruments with minimal ID-only PUCKs. When the instrument is plugged in, the SIAM node obtains the UUID. If the instrument has been registered using addService, the UUID may be used to find the appropriate instrument service content (e.g., JAR file).
The listServices utility makes it possible to know what ID-only instruments have been registered.

Options

Option Description Values
nodeURL IP address or host name of SIAM node e.g. 134.89.88.1, mooring.moos.mbari.org

Example output:

listServices localhost
00000000-0000-0000-0000-000000000000 - Elmo Louver Controller

Environment

The following environment variables must be set:

Variable Description Example
JAVA_HOME location of Java installation
/j9, /opt/java/jdk1.3.1, etc.
JAVA Path to JVM command For MMC
$JAVA_HOME/bin/j9

For Sun JVM (e.g. linux, win32, mac OS)
$JAVA_HOME/bin/java
SIAM_HOME top of the SIAM directory tree -
SIAM_CLASSPATH Path including all SIAM and supporting classes For MMC
$JAVA_HOME/lib/jclMax/classes.zip:$JAVA_HOME/lib/prsnlmot.jar:$JAVA_HOME/lib/RXTXcomm.jar:$SIAM_HOME/classes

For linux/win32
$SIAM_HOME/classes
